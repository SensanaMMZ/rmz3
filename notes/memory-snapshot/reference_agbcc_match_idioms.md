Two recurring agbcc codegen quirks in rmz3 decomp and the source idioms that match them. See also [[feedback-decomp-build-verify]].

**Slot-pointer register pattern (CSE build-then-copy).** When the original builds a struct slot address in a scratch reg, derefs to another reg, then copies the address to a callee-saved reg (`adds r0,r4; adds r0,#0xbc; ldr r1,[r0]; adds r5,r0,#0`), a named local `slot = (struct Entity**)((u8*)p + 0xbc)` produces the WRONG code (agbcc builds directly in r5). Fix: inline the full `*(struct Entity**)((u8*)p + 0xbc)` expression at every use (no named variable) — agbcc's CSE then keeps the address in a scratch and copies to the saved reg. Used for CapsuleCannon_Update, Claveker_Update.

**Prevent range-folding of consecutive int compares.** Original tests `mode[1]` against e.g. {2,3,5,4} as four separate `cmp;beq`. Writing `if (m==2||m==3||m==5||m==4)`, a switch, or `&&` all let agbcc fold the consecutive pair 2,3 into a range check `(m-2)<=1` (subs/lsls/lsrs/cmp/bls). Fix: emit each as its own statement — `if (m==2) goto skip; if (m==3) goto skip; ...` — so the fold pass can't see them as one boolean expr. Order of the gotos matches the asm cmp order.

**EXIT_BODY despawn register reuse.** When a despawn block clears the body (`body.status=0; prevStatus=0; invincibleTime=0; flags&=~COLLIDABLE`) the original often reuses a register already holding 0 (e.g. the `props[N]` value the guard just confirmed is 0, or a `status & DEAD` value). The EXIT_BODY macro writes literal 0 and gets a fresh `movs`; to match, skip the macro and write the fields manually using that loaded value: `(p->body).status = prop;` etc. (works when the value is provably 0 on that path — PurpleNerple_Update). If the path overrides the value (e.g. agbcc emits its own `movs r2,#0` because the masked value was nonzero on the taken branch — GeneratorCannon), this is unreproducible from C.

**bool8 vs s32 return at the test site.** `if (helper(p))` where the helper returns `bool8` makes agbcc zero-extend the result (`lsls#24`) before the branch. If the original has that `lsls#24`, declare the helper `bool8` (GeneratorCannon FUN_0808c3ec). If the original tests with a bare `cmp r0,#0` (no extend), declare/define the stub returning `s32` (Lamplort FUN_0806ce5c).

**Signed vs unsigned compare + branch direction.** `x & 3` on a signed `s32` compiles to a signed compare (`ble`/`bgt`); if the original uses unsigned (`bls`/`bhi`), make the operand unsigned: `(x & 3U) > 1`. Separately, agbcc keeps the *first-written* if-branch as the fall-through; if the original puts block B out-of-line, write B in the `else` (HanumachineObj_Update blink toggle).

**gOverworld.sea / load scheduling.** The sea Y-coord field `gOverworld.sea` (macro `SEA`, in overworld_terrain.h) is at offset 0x2C00C — when asm shows `ldr =gOverworld; ldr =0x2C00C; adds; ldr [r0]`, it's `gOverworld.sea`, NOT a raw `*(s32*)((u8*)&gOverworld+0x2C00C)` (the raw cast makes agbcc reassociate the constant onto the other comparison operand — wrong code). Two scheduling fixes seen in the sea-sink Updates (Shelluno, PantheonAqua): (1) to evaluate a field load *before* the other compare operand, hoist it to a local right before the `if` (`s32 sea = gOverworld.sea; if (sea > ...)`); (2) to get "compute address early, defer the load" (props[0] loaded after coord.x), use a *pointer* local: `s32* ppx = (s32*)((u8*)p+0xb4); ... cx - *ppx`. Declaration order of plain locals also sets load order / which register each lands in — reorder the declarations to match.

**bool8 guard merge — tail-merge instead of a variable.** When two if/else branches each produce a bool8 guard tested by a shared `if (guard) return;`, storing it in a `bool8 guard` local makes agbcc materialize it with `lsls#24; lsrs#24; cmp` (extra lsrs). The original keeps the raw helper result in r0 and tests with just `lsls#24; cmp`. Fix: drop the variable and write a separate inline `if (helper(p)) return;` in *each* branch — agbcc tail-merges the identical returns into one `lsls#24; cmp` with no lsrs (TileCannon_Update).

**Shared-store temp: use s32, not u8.** When two branches funnel a value into one shared store of a u8 field (e.g. `*t = *t-1` vs `*t = 0x60` merged via `b store; store: strb r0,[t]`), use a temp to force the merge — but declare it `s32`, not `u8`. A `u8 v` makes agbcc truncate at the assignment (`lsls#24; lsrs#24`, +4 bytes → ROM overflow); `s32 v` keeps the raw value and lets the final `strb` truncate, matching the original (Anubis_Update timer).

**Boss/element-slot Update family.** Many bosses share a shape worth recognizing: an alive check `if ((!dead && *(s16*)(p+0xa4)) || (gStageRun.missionStatus & 8)) goto alive; else DIE` (missionStatus is gStageRun+8; bit3=FAIL); an element-effect child slot at p+0xb4 or +0xc4 cleared on kill / spawned via `ApplyElementEffect(idx, &p->s, &sElementCoord)` with a shared store through a temp `e`; then a guard helper and a two-table dispatch (often the first table NULL-checked). Inline the slot expression to avoid an extra callee-saved push. Done: Cubit, Hanumachine, LocomoIF, Anubis.

**Redundant `& 0xff` before `& 1`.** When the original masks an incremented value `(work[2]+1) & 0xff` then `& 1` (two `movs;ands` pairs), writing `x & 0xff & 1` in one expression makes agbcc fold it to `x & 1`, and a `u8` cast gives `lsls#24;lsrs#24` instead. To get the literal `movs #0xff; ands; movs #1; ands`, split the mask into its own statement: `blink = work[2]+1; work[2] = blink; blink &= 0xff; if (blink & 1)` (CarrybeeG blink toggle). Struct-copy a Coord (`(p->s).coord = par->coord`) to get load-both-then-store-both instead of sequential load/store.

**Duplicated despawn blocks beat goto/merge+temp (tail-merge idiom).** When an Update has two despawn paths that converge to one shared tail in the asm (e.g. `_0808C678`: clear FLIPABLE, store flags, EXIT_BODY, SET_ROUTINE), that shared tail is usually agbcc *tail-merging two fully-duplicated `if(...){ ...; return; }` blocks* — NOT a hand-written `goto merge` with a carried `fl`/`u8` temp. The carry temp forces the half-cleared flags into the "wrong" register (e.g. `fl` lands in r1 so `ands r1,r0`; target keeps it in r0 via `ands r0,r1`) — an r0/r1 swap that neither hand-tuning nor the permuter could flip (GeneratorCannon stuck at score 70). Writing the two blocks out in full (`flags&=~DISPLAY; flags&=~FLIPABLE; body.status=...; ...; SET_ENEMY_ROUTINE(...); return;` duplicated) lets agbcc tail-merge the common suffix AND keep the flags value in r0 → match (GeneratorCannon_Update, `fac7cf7a`). The consecutive `flags&=~DISPLAY; flags&=~FLIPABLE;` still coalesce to one load + two ands; the per-path zero (`movs r2,#0` vs reusing sf) splits the merge point exactly as the asm shows. Combine with the `do{}while(0)` nudge below for any leftover (u8)-cast-into-home-reg materialization.

**`do{}while(0)` register-home nudge (permuter-found).** When a value must materialize *straight into its home (callee-saved) register* but agbcc computes it in a scratch and copies (`lsrs r0,..; adds r5,r0,#0` instead of `lsrs r5,r0,#0x18`), there's no direct C knob — but wrapping a *later* statement that uses the value in `do { ...; } while(0)` can perturb the allocator into emitting it directly, dropping the copy (and the alignment-pad it caused). This is what landed `Gallisni_Update` (do/while around the path-B `prevStatus = sf` write). Don't hand-guess these — let the permuter find them; see the recipe in [[project-open-todos]]. The permuter is now reliable for this whole allocation/scheduling class thanks to the `--disassemble=<func>` symbol-isolation scorer patch (`--no-prune` faithful base.c). See [[reference-decomp-tooling]].

**`do{ return; }while(0)` also flips early-return branch SENSE (permuter-found, blazin_tail onCollision `6a1bfdcb`).** Second use of the same lever: an early `if (cond) return;` where agbcc emits `bgt <return>` (branch-to-return) but the target wants `ble <shared-continue>; b <return>` (branch-to-continue, return as the fall-out). Plain `return;`, inverting the condition, and an empty-then/`else return;` all stayed `bgt`. Wrapping the early return — `if (cond) { do { return; } while (0); }` — made agbcc lay out the inverse (`ble` to the shared tail). The permuter found it at iter 264 with reorder weights (`perm_reorder_stmts`, `perm_sameline`, `perm_associative`, `perm_inequalities`); the bare-block `if(1){...}` it also emitted was NOT the trigger — only the `do{return;}while(0)` is. **Permuter-setup gotchas this run:** (1) use the Python that has `toml` (`AppData/.../Python313/python.exe`, not `py`); (2) `setup_fn.sh` needs an `.inc` — for an inline-asm NAKED, carve the body to a `.inc` + convert to `INCCODE` first (also dodges the candidate/stub name clash, since the awk only strips `.include`-style stubs); (3) the candidate is appended *after* preprocessing, so it must use already-expanded forms — no `UNUSED`/`FALSE`/`TRUE`/macros (write `0`/`1`); (4) GNU `field : value` designated initializers in data (e.g. `sCollisions`) break the permuter's pycparser — stub the offending array initializer to `= {{0}}` in `base.c` (the function being permuted doesn't read it).

**Bind global tables to locals to force early literal-load hoist.** A table-lookup function (`gMPlayTable[gSongTable[id].ms].info`) where the target loads BOTH table base literals (`=gMPlayTable`, `=gSongTable`) right after `push` — but clean C loads each at its use point (later, interspersed). Fix: bind each global array to a local at the very top, **in the target's load order**: `const struct MusicPlayer* mt = gMPlayTable; const struct Song* st = gSongTable; ... &st[id]; mt[...]`. agbcc then materializes the literals early. Matched `fadeoutSound` (`make compare` OK after it). Same load-scheduling family as the gOverworld.sea hoist and the declaration-order levers above.

**Two identical duplicated branches (cross-jump-failure) are NOT C-reproducible.** When the target has an `if(cond)` splitting into two BYTE-IDENTICAL blocks each with its OWN literal pool (e.g. `getSunkenLibRoomCoord`: both arms `&sSunkenLibRoom[*(u8*)(0x0202FFF4+idx)]`), writing the if with identical arms makes agbcc **tail-merge** them (13 vs target's 23 instr). The retail duplication is a cross-jumping failure no clean C triggers — leave as NAKED/dual-form.

**Split .inc header.** When splitting a `_p1.inc` into `_a`/`_b` for incremental decomp, the tail half loses the file header. 3-operand `adds r0,r0,r1` then errors "not supported in Thumb16". Prepend `.include "asm/macros.inc"` / `.syntax unified` / `.text` to the `_b` half.

**Predicate return shape — two returns, not `return cond`.** For a `bool8`/`s32` predicate whose asm is `cmp;beq L; movs r0,#0; b E; L: movs r0,#1; E: ...`, writing `return cond;` makes agbcc emit a compact set-from-flags form (or for ternaries an `adds r1,#1` increment trick — `kiss_080dac04`). To get the literal two-`movs`-with-branch shape, write two explicit returns: `if (cond) { return TRUE; } return FALSE;` (volteel FUN_080459d4, cubit_08054674). **Branch direction / fall-through is set by which value you make the early return:** agbcc keeps the *non-early* return as fall-through. If the asm falls through to value B (i.e. `beq`s to value A), write `if (test_for_A) return A; return B;` — e.g. kiss falls through to 0x2b7, so `if (counts[24]==0) return 0x2b6; return 0x2b7;` matched (the natural `if(counts[24]) return 0x2b7; else return 0x2b6;` gave the inverted `bne`). Invert the C test to flip beq/bne.

**Reuse the checked value instead of a literal zero (with a temp).** When a guard `if (field == 0) { other = 0; field++; }` stores 0 to another field and increments, the original often keeps the just-loaded `field` value in a register and reuses it for the `other = 0` store, then *re-reads* `field` for the increment. Writing `other = 0` (literal) makes agbcc either materialize a fresh 0 or copy via the wrong reg. Fix: load the field into a temp and assign that: `u8 m = (p->s).mode[2]; if (m == 0) { (p->s).d.y = m; (p->s).mode[2]++; }` — agbcc keeps `m` in r0 (the cmp reg), reuses it for the store, and re-loads mode[2] for the `++` (matches `str r0; ldrb r0,[..]; adds r0,#1`). Note: `other = (p->s).mode[2]` *without* the temp picks a different register (one-reg shift); the explicit `u8 m` temp pins it. (FUN_0807be20, beetank sub-function.)

**Minigame B-button check family (player/minigame, game/minigame).** Small `(struct Zero* z)` ZeroFunc table entries returning `bool8` (always `return TRUE`) that gate a mode switch on a button press: `if (<cond> && (gJoypad[0].pressed & B_BUTTON)) { (z->s).mode[1]=3; (z->s).mode[2]=0; } return TRUE;`. The condition is either `(z->s).mode[2] > 1` (harpuia FUN_080353e4, zero FUN_08036848 — byte-identical) or `((struct MinigameState*)(z->s).unk_28)->unk_04 == 1` (harpuia FUN_0803537c — `unk_28` is the parent `MinigameState*`, its s16 `unk_04` at offset 4 reads as `ldrsh`). Variant FUN_080354a4 sets `mode[3] = mode[2]` (reuses the mode[2] register) gated on `mode[2]==1 && *(s8*)((u8*)z + 0x71) > 3 && B_BUTTON`. Needs `#include "input.h"` (gJoypad is `struct KeyState gJoypad[2]`; `.pressed` is the offset-4 field = newly-pressed keys; `B_BUTTON`=0x2) and `#include "minigame.h"` for MinigameState. These funcs are often forward-declared `void` and used in the ZeroFunc table without a cast — change the decl to `bool8` AND add `(ZeroFunc)` at the table entry. **Open:** harpuia FUN_080353a8 (SetMotion(0xBE21)+SetDDP switch-init) needs the sCollisions index for `0x0835eaf8` resolved — it computed to sCollisions[0]+40 (= [1]+16), not a clean 24-byte struct base, so the SetDDP arg indexing needs care; left as asm for now.

**Stage-gfx StageLayer-callback family (clean vein).** `src/stage/<name>/landscape.c` files include `asm/stage_gfx/<name>.inc` and the `[LAYER_UPDATE/DRAW/EXIT]` table entries are small `(struct StageLayer* l, const struct Stage* stage)` callbacks. Reliable matches: **phase-gated one-shot inits** `if (l->phase == 0) { l->scrollPower.x = N; l->scroll.x = M; l->phase++; }` (FUN_0800f874); **gated draws** `if (l->phase > 1 && l->unk_0f != 0) DrawGeneralStageLayer(l, stage);` (FUN_0800f2c4); **window/blend reg clears** `gWindowRegBuffer.dispcnt &= ~DISPCNT_WIN1_ON; gWindowRegBuffer.winin[2] |= 0xe;` (FUN_08015c40, FUN_0800d264 — need `#include "gpu_regs.h"`); **stage-state stores** via the file's `#define STAGE (gOverworld.work.<stage>)` macro: `STAGE.unk_018 = a; ...` (FUN_08016094). **StageLayer offsets are bigger than the header comments suggest** — `phase` lands at **0xe** (not 0x6: the `fn` routine field + `type` consume more), `unk_0f`=0xf, `scrollPower.x`=0x44, `scroll.x`=0x4c, `bgIdx`=0x5c; just write the field names and let the build resolve them, diff2 confirms instantly. **BGCNT-priority idiom (matched):** `BGCNT16(l->bgIdx >> 4) = (BGCNT16(l->bgIdx >> 4) & 0xFFFC) | 3;` (resistance_base rbase_0801666c — plain `bgIdx >> 4`, no cast). **HOLDOUT:** missile_factory FUN_0800f840 wants `(bgIdx<<16>>0x14)<<1` (a `(u16)` truncation) AND partially hoists only the `<<16` out of the `if(phase==0)` guard — no clean C reproduces that partial-CSE (tried `(u16)` cast → right shifts wrong regs; `u16* bgcnt=&BGCNT16(...)` hoist → materializes whole address too early). Left as asm.

**Pointer-temp forces address-built-before-zero.** When a block does one byte store to a far field (`*(u8*)(p+0xc1)=0x3c`) then several stores of 0 (`d.y=0; mode[1]=0; mode[2]=0`), the original often builds the 0xc1 *address* into a reg first, *then* `movs r1,#0`. Writing the store plainly (`*(u8*)((u8*)p+0xc1)=0x3c`) makes agbcc emit the zero first (reusing r0) — wrong order, byte diff even though same size. Fix: bind the far address to a pointer local *and* the zero to an `s32` local, both declared before the stores: `u8* t=(u8*)((u8*)p+0xc1); s32 zero=0; *t=0x3c; d.y=zero; mode[1]=zero; mode[2]=zero;`. The pointer-temp makes agbcc materialize the address into its reg before the zero (FUN_080795b8, shelluno). Just the `s32 zero` temp alone is *not* enough — it still puts `movs r1,#0` first.

**Face-Zero / face-attacker BodyFunc family.** A common `BodyFunc(struct Body* body)` on-hit handler: `if (body->hitboxFlags & 1) { self = (Boss/Enemy*)body->parent; *(u8*)(self+OFF) = (A.coord.x </> self.coord.x); }` where the result byte is the facing direction. The attacker A is either `pZero2` (CopyX_OnDamage→0xc4, FUN_0806465c pantheon_guardian→0xb8) or `body->enemy->parent` (anubis FUN_080500c8→0xcc, hanumachine FUN_0805d568→0xbc). **Deref `body->parent` lazily *inside* the `if`** (not hoisted) — the original loads `[body+0x2c]` after the `beq`, so hoisting `self` adds an instruction and shifts the next symbol 4 bytes (caught only by full `make compare`, the isolated disasm looks fine). The compare branch (`bge` vs `ble`/`blt`) follows operand order; pick `A.x < self.x` vs `A.x > self.x` to match. pZero2 needs `#include "zero.h"` (safe — anubis uses it).

**Offset-table base load order — pointer temp assigned AFTER the preceding call.** Indexing a sub-view of a table, `SetMotion(&p->s, (&sMotions[5])[work[2]])`, makes agbcc load the *index* first then the base literal (`=sMotions+10`) — wrong order vs a target that loads the base first (and the plain `sMotions[work[2]]` already loads base-first, so only the `&table[N]` offset form is affected). Writing `sMotions[5 + work[2]]` instead leaves a stray `adds r0,#5` (the +5 stays in the index). Fix: a pointer temp, but **declare it uninitialized and assign it after any preceding call** — `const motion_t* m; SetDDP(...); m = &sMotions[5]; SetMotion(&p->s, m[work[2]]);`. Declaring+initializing it before the call (or at function/case top) makes it live across the call → callee-saved → an extra `push {r5}`; assigning it after the call keeps it in a scratch and loads base-first. Matched FUN_080a5e00 (`dd04ca65`). Same `&inner` lever as the camera-viewport split below.

**Subtraction-with-constant reassociation — bind the sum to a temp.** `a - (b + K)` (K a literal) gets reassociated by agbcc into `(a - K) - b`, materializing a `-K` literal from the pool (extra `ldr [pc]` + the wrong operand order). The target keeps `b + K` as a live subexpression: load `b`, `adds K`, load `a`, `subs`. Fix: compute the sum into its **own** temp first — `s32 t = (p->s).coord.y + 0x1000; dy = (pZero2->s).coord.y - t;` — so the constant can't fold across the subtract. Bonus: giving the sum a *distinct* temp (not reusing `dy`) also pinned `pZero2` into the target's register and killed a stray `adds rX,rY,#0` copy in the follow-on squaring — the 179/179 lever for FUN_0809f64c (`60c29d72`). Same family as the gOverworld.sea raw-cast reassociation below.

**Materialize-a-zero-early lever: function-scoped temp assigned AFTER the call/op (permuter-found).** When the target materializes a constant (e.g. `work[3] = 0`) into a register *early* — right after a libcall/`__divsi3` — and holds it across later stores, but agbcc-from-clean-C schedules the `movs` late (fused with the `strb`, in the reused scratch): declare the temp at **function scope** uninitialized (`s32 zero;`), assign it `zero = 0;` at the **exact early point** (after the division), and store `field = zero;`. The post-op assignment pins the materialization point; the function scope + separated decl/assign stops agbcc const-folding it back to the use site (a *case-scoped* `s32 zero = 0;` const-folds away and reschedules late — verified failing). Matched FUN_080a25f8 (`ae37e840`). This is a permuter `perm_pad_var_decl`+placement find — for these "value-materialization-point" cliffs (all-or-nothing score, no gradient), set `[weight_overrides]` cranking the nudge/pad passes (`perm_pad_var_decl`, `perm_ins_block`, `perm_empty_stmt`, `perm_duplicate_assignment`, `perm_reorder_stmts`, `perm_split_assignment`) — found at iter 90 vs 607 fruitless at default weights. See [[project-open-todos]] item 5.

**Camera-viewport address split — explicit `&` on the inner struct.** Off-screen kill checks read `gStageRun.vm.camera.viewport.x` (whole address = `gStageRun+0x120`, > Thumb word-load imm range). The target keeps the **base literal `=gStageRun+232` (=0xE8, the address of `.camera`) + a `ldr [r0,#0x38]` field offset**, but a plain `gStageRun.vm.camera.viewport.x` *or* a raw `((s32*)((u8*)&gStageRun+232))[14]` cast both FOLD to one `=gStageRun+0x120; ldr [r0,#0]`. Fix: take the address of the inner struct explicitly — `(&gStageRun.vm.camera)->viewport.x` — so agbcc materializes `&camera` as the symbol+232 literal and applies `viewport.x` as the `[+0x38]` MEM offset (repo idiom; also `(&gStageRun.vm.camera)->viewport.y` in volcano_bomb, `(&gStageRun.vm.camera.viewport)->x` in cyberelf/unk_0). Confirmed on FUN_080b17dc (`e657f3db`). Combine with the duplicated-despawn idiom below: that function's two `work[2]` branches each carry a full `flags&=~DISPLAY; flags&=~FLIPABLE; EXIT_BODY(p); SET_ROUTINE(ENTITY_DISAPPEAR)` so cse reuses the `work[2]==0` reg as the EXIT_BODY zero in the ==0 path. Same recipe matched FUN_0809ceb0 (shotcounter bullet, `f91ecda1`) first-try: duplicate the EXIT_BODY+PlaySound+CreateSmoke die tail into both `body.status & FLAG` branches.

**`K - mem` subtraction: load the operand into the accumulator first.** `return CONST - z->field;` makes agbcc materialize CONST into a fresh scratch reg, COPY it to the result reg, then subtract (`movs r2,#..; lsls; adds r0,r2,#0; subs r0,r0,r1` — extra copy). The target materializes CONST straight into the result and reloads the field first. Fix: two statements through one accumulator — `s32 v = z->field; v = CONST - v; return v;`. Loading the field into `v` first forces the field-reload to precede the constant materialization AND coalesces CONST into the result register (no copy). Solved the long-standing `GetSplitHeavenSpeed` (`0x100 - z->slow`, dev comment "such simple code but won't match, why?!", `9ade4323`). NB the plain accumulate `v = CONST; v -= field;` fixes the copy but schedules CONST *before* the field load (order flips) — must seed `v` with the field, not the constant.

**Pointer-walk down-counter loop (m4a volume family) — split count from counter + guard-scope the pointer.** A loop over a global table that the asm walks by POINTER (`adds r5,#0xc`) and counts DOWN (`subs r4,#1; cmp; bne`) with a pre-guard (`cmp; beq end`). Three levers, all needed: (1) walk a pointer, not an index — `const struct MusicPlayer* mpt = gMPlayTable; ...mpt->info...; mpt++;` (an index `tab[i]` gives lsls+add addressing, wrong); (2) **scope the pointer INSIDE the entry guard** so its table-base load lands after the `if`, not hoisted before it — `if (n != 0) { const T* p = tab; ...; }`; (3) **split the loop count from the loop counter** — `u32 n = COUNT; if (n != 0) { ...; u32 i = n; do {...} while (--i != 0); }` — the guard tests `n` (scratch r0), and `i = n` emits the `adds r4,r0,#0` copy into the callee-saved counter that the target has. Writing `i = COUNT; if (i)...` (one var) puts the value straight in r4 with no copy (1 instr short). Count source matters too: `NUM_MUSIC_PLAYERS = (u16)gNumMusicPlayers` is an ABSOLUTE LINKER SYMBOL (=15), so it pool-loads + `lsls#16;lsrs#16` casts — the plain literal `MUSIC_PLAYER_LENGTH`(15) would `movs #15` instead. Matched TurnDownBGM/TurnUpBGM (`f5d683cf`).

**Bind a union/sub-struct to one local pointer to SHARE the base register (two near-offset fields).** When a function reads two fields of the same far sub-struct — e.g. `w->props.buster.z` (offset 0xb4) and `w->props.buster.unk_c3` (0xc3) — agbcc-from-naive-C builds TWO separate base pointers (`w+0xb4` and `w+0xc3`), one extra `adds`. The target computes `&w->props` ONCE and uses `[base]` + `[base+0xf]` displacement. Fix: bind the sub-struct to a single local pointer up front and access both fields through it: `struct Buster_b4* b = &w->props.buster; z = b->z; if (b->unk_c3) { b->unk_c3--; ... }`. agbcc then keeps one base reg + displacement, matching. **Pointer-temp before a variable index forces `(base+const)` first, not `(index+base)+const` (CielMinigameEnemy2_Update, `10a3e39c`).** Indexing a far table with a variable — `*((u8*)e + 0xDE0 + work[0])` or `((u8*)e + 0xDE0)[work[0]]` — makes agbcc reassociate to `(work[0] + e) + 0xDE0`: it loads the index, adds the base, then adds the constant LAST (reusing the base's register for the `0xDE0` since the base is now dead). **Signed-char field: `(s8)` cast gives `ldrsb` on value-use but the RMW stays `ldrb` (readKeyInput, `b2f36d5e`).** When the asm loads a `u8`-declared field SIGNED for a comparison (`movs r0,#0; ldrsb r0,[r2,r0]; cmp r0,#0`) but UNSIGNED for a read-modify-write of the same field (`ldrb; subs; strb`), the field is semantically a signed char: GCC 2.9 sign-extends on value-use (`x == 0`, needs the s8 value) but uses `ldrb` for `x--` (truncates on store, sign irrelevant). **Packed-bitfield read WITHOUT a bitfield struct: explicit unsigned shifts + temps (CRACKED IsElfBreeder, `79296cae`).** When the target reads a packed field with `lsls #(32-hi); lsrs #(32-width)` (agbcc's unsigned bitfield extract) but the field lives in a SHARED `u8`/`u32` type you can't safely convert to a bitfield struct (its data initializer / other readers depend on the plain layout), reproduce the extraction inline: a field at bits `[lo, lo+w)` of value `x` = `((u32)x << (32-lo-w)) >> (32-w)`. E.g. bits 0-2 → `((u32)x << 29) >> 29`; bits 3-4 of a word → `(word << 27) >> 30`; bits 3-5 → `((u32)x << 26) >> 29`. The `(u32)` cast is REQUIRED (forces `lsrs` not arithmetic `asrs`; without it the u8/int promotion sign-extends). Two scheduling levers finished the exact match: (1) when two fields are extracted and compared (`extractA != extractB`), agbcc emits BOTH left-shifts before BOTH right-shifts — bind each `<<` to its own temp (`u32 a = x<<29; u32 b = word<<27; if (a>>29 != b>>30)`) so the `>>`s defer; inline `(x<<29>>29) != (word<<27>>30)` does the `>>` eagerly (2 instr off). (2) To emit the loop counter init (`movs r4,#0`) BEFORE the hoisted table-base pool loads, initialize it at its declaration ABOVE the pointer locals (`u8 i = 0; const T* info = tbl; ...; for (; i < N; i++)`) not in the `for` header. **State-machine dispatch: jump table vs switch vs if/else-if (2026-07-12, whole actor.c + batring/ciel_computer vein).** Branchy `mode[1]`-dispatched `*_Update` state machines match CLEANLY (they were mischaracterized as hard). Pick the C construct by the asm's dispatch shape: (a) a **dense jump table** (`cmp #N; bhi default; lsls #2; ldr =tbl; mov pc, r0`) ⇒ write a `switch` over cases 0..N with `break`/fall-through as the asm shows; agbcc reproduces the table exactly. An **empty trailing `case N: break;` extends the table's range** — agbcc sizes the table by the max case label, so a hole at the top (e.g. the asm's table has 8 entries but your max case is 6) needs a `case 7: break;` to get `cmp #7`. Fall-through between cases (no `break`) is fine. agbcc auto tail-merges shared `SetMotion(m); mode[1]++` / bare `mode[1]++` case tails. (b) **Exactly 2–few cases via sequential compares** (`cmp #0;beq;cmp #1;beq;b default`) ⇒ ALSO a `switch`, NOT `if(m==0){}else if(m==1){}` — the if/else-if defers the 2nd compare to AFTER the 1st block (wrong layout); the switch emits both compares upfront. (c) **function-pointer TABLE call** (`ldr =sTbl; ldrb idx; lsls #2; adds; ldr; bl _call_via_r1`) ⇒ `sTbl[(p->s).mode[1]](p)` with `sTbl` a file-scope `const EnemyFunc[]`. Common sub-idioms in these: `if ((p->s).scriptEntity->flags & BIT)` (scriptEntity @0x18, its flags @0x9); `(p->s).motion.state` @0x73; `SET_XFLIP(&p->s, 0)` inside `if((u8)work[N]==0)` reuses the known-0 reg; **shared flip-value local** — for `SET_XFLIP(p,V)` then `CreateParticle(...,(flags>>4)&V)`, declare `bool8 isRight=1;` and use it in BOTH so the constant coalesces (else `1` materializes twice, +1 instr).

**Ternary condition for a single shared block (Batring_Update, `c0463698`).** An either-side / abs-value range test whose two sign-branches run the SAME block (retail cross-merges them) matches when written as ONE ternary condition: `if (diff>0 ? diff>K : x-zx>K) { <shared block>; }`. The natural nested-if form (`if(diff>0){if(diff>K){B}}else if(x-zx>K){B}`) with two identical `{B; return;}` blocks is ~1 instr TIGHTER but lays the branch out the other way (`ble normal` inline-die vs retail's `bgt die; b normal`). **Macro-arg-as-variable when a macro re-uses its arg (CielComputer_Update, `3f5be729`).** A graphic-load macro like `STATIC_GRAPHIC_LOADER(fn, n)` substitutes `n` at several sites (`sizeof*n` for the offset AND `wStaticGraphicTilenums[n]`). **Interleaved member-writes vs struct-copy for a small Coord store (FUN_0801f714, `a93228fa`).** Copying a 2-word Coord into a raw slot: `*(struct Coord*)dst = *c;` gives load-both/store-both (ldr,ldr,str,str); retail often does interleaved member writes (ldr,str,ldr,str). **Named field vs raw-cast for a LARGE struct offset — folding (updateProjectile2, `be8e58bc`).** Accessing a global-struct field whose offset exceeds the ldr immediate (>4095, e.g. `gOverworld.sea` @0x2C00C or `gOverworld.state[0]` @0x2D024) emits TWO pool constants: `ldr =gOverworld; ldr =0x2C00C; adds; ldr [r]`. **★★ COPY-PIN cracks the "redundant register spreading" / reload-double-home residual that the coalescer theorem called C-UNREACHABLE (2026-07-13 — corrects a multi-session conclusion).** The trick: agbcc coalesces a plain copy `y = x` (same value → one register), so you can't emit a redundant copy from source — UNLESS you pin source and dest to DIFFERENT physical registers: `register T x asm("r0") = val; register T y asm("r5") = x;` forces `mov/adds r5,r0` (uncoalesceable, no `long long`, no dead `asr`). (2) **pins are LOOSE** (agbcc only guarantees the register at asm-operand points) — a value that's live-for-store may still be compared directly instead of via its pinned copy (beetank's `a`), and pins on many interacting values cascade; and **pycparser rejects `register asm`, so a copy-pinned base CANNOT seed decomp-permuter** (the pins are a hand-only tool). Net: copy-pin is the right first move on any redundant-copy/double-home residual, and it fully cracks the SINGLE-residual cases (ResetCharTiles's base-copy) even if a multi-residual function (beetank) has other ties left.
